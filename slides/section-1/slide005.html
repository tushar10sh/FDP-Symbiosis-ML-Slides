Types of ML Systems

Slide 5 / 15

ML systems are classified along three independent axes — these criteria are not mutually exclusive; a real system can combine all of them.

Axis 1 — Supervision

How much human labelling during training?

Supervised

Training data includes desired labels (answers). Used for classification & regression.

Unsupervised

No labels — system discovers structure on its own. Clustering, dim. reduction.

Semisupervised

Mostly unlabelled data + a few labels. Google Photos face grouping.

Reinforcement Learning

Agent acts, gets rewards/penalties, learns an optimal policy. AlphaGo.

Axis 2 — Learning Style

Can the system learn incrementally?

Batch (Offline) Learning

Trained on all available data at once. Can take hours. Then deployed with no further learning. Must retrain from scratch for new data.

Stable Resource-heavy

Online (Incremental) Learning

Trained incrementally in mini-batches. Can learn on the fly. A key parameter is the learning rate: high = fast adapt but forgets; low = stable but slow.

Adaptive Out-of-core capable

Axis 3 — Generalisation

How does it predict on new data?

Instance-Based

Memorises training examples. Generalises by comparing new data to known examples using a similarity measure. Simple example: k-NN.

k-Nearest Neighbours

Model-Based

Builds a model from training examples, then uses the model to make predictions on new data. Most practical ML systems use this approach.

Linear Regression Neural Networks

Example: A state-of-the-art spam filter can be online (incremental), model-based (deep neural network), and supervised — all three axes combined.
