Unsupervised Learning

Slide 7 / 15

Core Idea

Training data is unlabelled — the system tries to learn structure without a teacher. The goal is to discover hidden patterns.

Clustering

Group similar instances together — the algorithm finds those connections without being told which group each instance belongs to.

Algorithms:

  • K-Means
  • DBSCAN
  • Hierarchical Cluster Analysis (HCA)

Example: Blog visitors split into groups — 40% male comic fans, 20% young sci-fi readers.

Anomaly Detection

Detect unusual instances — shown mostly normal examples during training, so it learns what "normal" looks like.

Algorithms:

  • One-class SVM
  • Isolation Forest

Also: Novelty detection — find new instances different from all training instances (requires very clean training set).

Dim. Reduction

Simplify data without losing too much information. Merge correlated features into one (feature extraction).

Algorithms:

  • PCA, Kernel PCA
  • LLE, t-SNE

Tip: Reduce dimensions before feeding to supervised ML — faster, less memory, often better performance.

Association Rules

Dig into large datasets and discover interesting relations between attributes automatically.

Algorithms:

  • Apriori
  • Eclat

Example: Supermarket sales reveal: BBQ sauce + chips → steak. Place items together!

Visualisation Algorithms — a special case of unsupervised learning

Feed complex unlabelled data → get a 2D or 3D representation that can be plotted. These algorithms try to preserve the data's structure (e.g., keep separate clusters from overlapping). Helps you understand how data is organised and spot unexpected patterns.

t-SNE is particularly popular: it can reveal semantic clusters even for image datasets (animals vs. vehicles, etc.).
