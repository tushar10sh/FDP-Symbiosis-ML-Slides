Machine Learning Approach

1

Study the problem

Manually study what spam looks like — certain words, patterns, senders.

2

Write detection rules

Code a rule for every pattern found. Flag email if enough rules match.

3

Test & repeat

Evaluate, find errors, add more rules. Repeat until good enough to launch.

Problems

  • Long list of complex, fragile rules — hard to maintain
  • Spammers adapt (e.g., "4U" → "For U") — rules need constant updates
  • Does not scale to thousands of new patterns
1

Feed labelled data

Provide thousands of emails already marked spam or ham (the training set).

2

Train the algorithm

The model automatically learns which words and patterns predict spam.

3

Evaluate & retrain

Measure performance, gather new labelled data, retrain as needed.

Benefits

  • Shorter, simpler programs — no hand-coded rules needed
  • Adapts automatically — re-detects "For U" without developer input
  • Can also reveal unknown patterns (data mining)

VS

When ML really shines

Problems needing lots of hand-tuning
Complex problems with no known algorithm
Fluctuating environments requiring adaptation
Gaining insights from complex data
