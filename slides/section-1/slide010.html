Instance-Based vs. Model-Based Learning

Slide 10 / 15

How does the system generalise? Most ML tasks require making good predictions on new, unseen instances.

Instance-Based Learning

The system learns examples by heart, then generalises to new cases by comparing them to learned examples using a similarity measure.

Spam Filter Example

A basic instance-based filter flags emails identical to known spam. A smarter version counts shared words — flags an email if it has many words in common with known spam (similarity measure).

k-Nearest Neighbours (k-NN)

Find the k most similar training instances and assign their majority class (or average value) to the new instance. Simple but powerful.

Pros

  • No training phase
  • Naturally adapts as new data added

Cons

  • Slow prediction (compares all instances)
  • Memory scales with training size

Model-Based Learning

Build a model from the training examples, then use that model to make predictions on new data. The model generalises the patterns learned from training.

The Workflow

  1. Study the data — explore & visualise
  2. Select a model — choose algorithm & architecture
  3. Train the model — run algorithm to fit parameters
  4. Apply the model — predict on new instances

Performance Measures

  • Utility / Fitness function: Measures how good the model is (maximise)
  • Cost function: Measures how bad it is (minimise). Linear Regression minimises distance between predictions and training examples.

Pros

  • Fast predictions
  • Compact — only need model params

Cons

  • Requires training phase
  • Must choose model carefully

The true goal of ML is not to perform well on the training data but to make good predictions on new, unseen instances. This is called generalisation.
