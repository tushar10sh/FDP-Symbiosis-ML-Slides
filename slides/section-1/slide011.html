Case study: Does money make people happier? Using GDP per capita to predict life satisfaction — a worked Linear Regression example.

The Model-Based ML Workflow

1Study the data

Download OECD Better Life Index + IMF GDP data. Join, sort by GDP, explore for trends.

2Select a model

Life satisfaction appears to increase linearly with GDP. Select a Linear Model with one predictor (GDP per capita).

3Train the model

Run Linear Regression algorithm — minimises the cost function (distance between predictions and training data). Finds optimal: θ₀ = 4.85, θ₁ = 4.91 × 10⁻⁵.

4Make predictions

Apply the trained model to Cyprus (GDP ~$22,587): predicted life satisfaction ≈ 5.96. Actual: 6.0 — close!

GDP per capita vs. Life Satisfaction

The Linear Model (Equation 1-1)

life_satisfaction = θ₀ + θ₁ × GDP_per_capita

θ₀ = bias (intercept) — 4.85
θ₁ = slope — 4.91 × 10⁻⁵
Model selection: choosing type + architecture

Alternatively, a k-NN regressor on the same data (k=3) also predicts ≈5.77 — this is instance-based, no model parameters needed.

The word "model" can mean: a type of model (Linear Reg.), a specified architecture, or a final trained model — context matters!
